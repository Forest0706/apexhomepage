# Apex Toys 官网项目 — 工时预估与进行表

**项目**：Apex Toys Headless 官网（Hydrogen + Shopify + Oxygen）  
**协作配置**：产品经理（需求与验收） + 全栈工程师（开发、对接、测试、UI）

---

## 一、项目范围与当前状态

| 项目 | 说明 |
|------|------|
| 技术栈 | Shopify Hydrogen（Remix + React）、Oxygen 部署 |
| 已完成 | 品牌首页、About 页、全局布局、购物车/账户/搜索等基础能力、本地开发环境 |
| 待完成 | 对接真实店铺、内容与分类、品牌完善、部署上线及验收 |

---

## 二、工时预估（产品经理 + 全栈工程师配合）

| 阶段 | 工作内容 | 预估工时 | 说明 |
|------|----------|----------|------|
| 1. 店铺对接 | Shopify 后台配置、Headless 渠道、API 凭证、本地/部署环境变量 | 1～1.5 人天 | 含账号与权限准备、联调 |
| 2. 内容与数据 | 真实商品与分类（Collection）配置、首页/列表数据源对接与展示 | 1.5～2 人天 | 依赖后台商品与分类就绪 |
| 3. 品牌与 UI | Logo、配色与字体微调、About 页内容与版式、必要组件样式统一 | 1.5～2 人天 | 按确认的文案与素材推进 |
| 4. 部署与上线 | 代码仓库与 Oxygen 部署配置、环境变量与域名（若需）、上线检查 | 1～1.5 人天 | 含一次完整回归 |
| 5. 验收与修改 | 客户验收、问题修复、小范围需求调整 | 1～2 人天 | 按实际反馈迭代 |
| **合计** | — | **6～9 人天** | 按 2 人协作、正常排期估算 |

*以上为在需求与素材及时到位、无重大范围变更前提下的预估；若增加独立专题页（如 ARCTECH 专题）或复杂定制，另行加 1～3 人天。*

## 六、假设与风险

- **需求与素材**：商品信息、分类、文案、Logo 等由客户/产品经理在约定时间内提供，延期可能影响总工期。
- **范围**：当前工时未含多语言、复杂营销规则、独立 App 或深度定制；若有新增范围需单独评估。
- **环境**：以客户提供或确认的 Shopify 店铺及权限为准；环境或权限不足导致的阻塞需双方协调解决。
