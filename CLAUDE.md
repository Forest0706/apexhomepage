## Project Overview

Apex Toys Storefront — a headless Shopify storefront for a Japanese figure/collectible shop, built with Shopify Hydrogen (Remix + React). The site is in Japanese (locale: JA/JP) and uses a light minimal aesthetic with stone/neutral color palette.

## Development Commands

```bash
npm run dev           # Start dev server with codegen (port 3000)
npm run build         # Production build with codegen
npm run preview       # Preview production build
npm run lint          # ESLint
npm run format        # Prettier format
npm run format:check  # Prettier check
npm run typecheck     # TypeScript check (run after build)
npm run e2e           # Playwright end-to-end tests
```

**Note:** `typecheck` requires a successful `build` first because GraphQL codegen must generate types.

## Architecture

### Stack

- **Framework:** Shopify Hydrogen ( Remix v2 + React 18), deployed to Shopify Oxygen
- **Styling:** Tailwind CSS v3 with CSS custom properties for the Apex theme
- **GraphQL:** Shopify Storefront API + Customer Account API; types auto-generated by `@shopify/hydrogen-codegen` (runs via `shopify hydrogen dev/build --codegen`)
- **Path alias:** `~/*` maps to `app/*`

### Key Files

- `server.ts` — Entry point for Oxygen worker; creates storefront client, customer account client, cart handler, and session
- `app/root.tsx` — Remix root layout, global loader (shop data, menus, cart, SEO), error boundary, Analytics provider
- `app/components/PageLayout.tsx` — Header (desktop/mobile) + Footer with APEX TOYS branding
- `app/routes/($locale)._index.tsx` — Homepage: Hero banner, featured products grid, series showcase, about section, newsletter

### Data Flow

- **Shopify API data:** Route loaders query Storefront API via `context.storefront`, return with Remix `defer()` for streaming
- **Fallback data:** `app/data/localProducts.ts` provides mock products when API fails (dev mode or demo credentials)
- **Translations:** `app/data/translations.ts` holds all Japanese UI text (keyed by `ja`)
- **GraphQL queries:** Centralized in `app/graphql/` — `HomepageQueries.ts`, `ProductQueries.ts`, `FeaturedProductsQuery.ts`, etc.
- **Fragments:** Shared in `app/data/fragments.ts` (e.g., `PRODUCT_CARD_FRAGMENT`)

### Route Structure

All routes use `($locale)` dynamic segment pattern. Key custom pages beyond the Hydrogen defaults:

- `($locale).about.tsx` — About page
- `($locale).purchase-guide.tsx` — Purchase guide
- `($locale).after-service.tsx` — After-sales service
- `($locale).faq.tsx` — FAQ
- `($locale).featured-products.tsx` — Featured products

### Theming

CSS custom properties defined in `app/styles/app.css` under `:root` — colors like `--apex-bg`, `--apex-accent`, `--apex-text`, etc. Tailwind config in `tailwind.config.js` maps these to utility classes (e.g., `bg-apex-bg`, `text-apex-text`).

### Selling Plans (Pre-orders)

Products can have `requiresSellingPlan` and `sellingPlanGroups` for pre-order functionality. GraphQL queries in `HomepageQueries.ts` and `ProductQueries.ts` include selling plan fields. The `checkoutCharge` type determines payment structure (PERCENTAGE vs PRICE).

## Codegen

GraphQL types are generated to `storefrontapi.generated.d.ts` and `customer-accountapi.generated.d.ts`. The codegen is triggered automatically by `shopify hydrogen dev` and `shopify hydrogen build`. Schema comes from `node_modules/@shopify/hydrogen/storefront.schema.json`.

Config is in `.graphqlrc.yml` — two projects: `default` (storefront) and `customer-account`.

## Known Issues

- **Node version:** `.nvmrc` says v18 but `package.json` engines require >=20. Use Node 20+.
- **Demo mode:** `.env` uses `mock.shop` by default; real store requires updating `PUBLIC_STORE_DOMAIN` and `PUBLIC_STOREFRONT_API_TOKEN`
- **Dev fallback:** `root.tsx` has a fallback layout when API fails in development mode

## CI

GitHub Actions runs: lint, format check, typecheck (after build), and e2e tests. Build requires `SHOPIFY_HYDROGEN_FLAG_LOCKFILE_CHECK=false`.
