# AGENTS.md - Apex Toys Storefront (Shopify Hydrogen)

## Dev Commands

```bash
npm run dev          # Dev server (codegen runs automatically via shopify CLI)
npm run build        # Production build (codegen runs automatically)
npm run preview     # Preview production build
npm run lint         # ESLint
npm run format       # Prettier format
npm run format:check # Prettier check
npm run typecheck    # TypeScript (run after build)
npm run e2e          # Playwright
npm run e2e:ui       # Playwright UI mode
```

## CI

4 jobs run in parallel: `lint`, `format`, `typecheck`, `e2e`.

- `typecheck` job runs `build` then `typecheck` (order matters)
- Build requires `SHOPIFY_HYDROGEN_FLAG_LOCKFILE_CHECK=false`

## Known Issues

- **Node version conflict**: `.nvmrc` says `v18`, but `package.json` engines require `>=20.0.0`. Use Node 20+.
- **CI `.nvmrc` mismatch**: CI uses `.nvmrc` (v18) while `package.json` specifies >=20. CI will use v18 unless fixed.
- **CLI manifest check**: CI's `typecheck` job has a step checking `packages/cli/oclif.manifest.json` which doesn't exist in this project. That step will always fail.

## Architecture

- Shopify Hydrogen + Remix (React), deployed to Shopify Oxygen
- GraphQL types auto-generated by `@shopify/hydrogen-codegen` (triggered by `shopify hydrogen dev/build --codegen`)
- Path alias: `~/*` → `app/*`
- Routes use dynamic `(locale)` grouping: `app/routes/($locale).*.tsx`
- Global layout: `app/components/PageLayout.tsx`
- Entry point: `app/root.tsx`

## Env Variables

Required in `.env`:

- `PUBLIC_STORE_DOMAIN`
- `PUBLIC_STOREFRONT_API_TOKEN`

Currently connects to `mock.shop` for demo. Update `.env` with real store credentials to switch.
